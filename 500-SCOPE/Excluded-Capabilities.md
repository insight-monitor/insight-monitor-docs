# Excluded Capabilities

## No intent certainty
The system does not determine real user intent. It produces probabilistic estimates based on available signals. Certainty is never assumed.

## No efficiency measurement
Efficiency is not a universal observable. The system lacks external references to measure output quality, business value, or optimal workflow. It cannot label activity as efficient or inefficient in an absolute sense.

## No task success validation
The system can detect probable progression (app switches, document saves, navigation patterns) but cannot confirm whether a task was successfully completed. "Success" requires external validation the system does not have access to.

## No covert surveillance
Transparency is a hard requirement. The user must know monitoring is active, what signals are collected, and have the ability to review and contest inferences.

## No replacement of human judgment
Inferences are inputs for managers, not verdicts. The system does not make employment decisions.

## No cross-device coverage
Activity on unmonitored devices (personal phones, physical whiteboards, paper) is invisible. The system's view is limited to the monitored endpoint.

## No offline/thinking detection
The system cannot reliably distinguish deep thinking from inactivity using screen data alone. Reading, contemplating, and being away from keyboard produce similar signals.
