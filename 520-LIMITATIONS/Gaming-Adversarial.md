# Gaming and Adversarial Behavior

The system is not immune to manipulation. Users may adapt to the inference model.

## Known gaming vectors
- Leaving productive-looking windows visible while doing non-work on a second device
- Shaping visible workflows to look productive
- Triggering context cues that bias classification (e.g., opening documentation while doing unrelated work)
- Exploit blind spots in screenshot intervals

## Limit
No monitoring system can fully prevent gaming. The system should surface evidence for its inferences so that patterns of gaming are detectable rather than hidden.
